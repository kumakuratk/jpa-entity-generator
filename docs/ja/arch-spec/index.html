アプリケーションアーキテクチャ

アプリケーションの挙動

jpa-entity-generatorはバッチアプリケーションである。 jpa-entity-generatorを実行するとDBからメタデータ(テーブル定義、カラム定義、キー情報等)を読み取り、それらにアクセスするためのJPA Entityのjavaソースファイルを生成する。 jpa-entity-generatorが提供する実行形態は、javaコマンド(Uber jar)とMaven Pluginである。

jpa-entity-generatorの挙動を以下に示す。

Figure 1. アプリケーションの挙動

  1. 利用者がjpa-entity-generatorを実行する。

  2. jpa-entity-generatorは設定ファイルを読み込む。利用者は予め設定ファイルに、DBへの接続情報やJPA Entityの生成に関する挙動を設定を定義する。

  3. jpa-entity-generatorはDBからメタデータを読み込む。

  4. jpa-entity-generatorはメタデータを元にJPA Entityのjavaソースファイルを出力する。

jpa-entity-generatorが使用する主要なJavaの機能・サードパーティー製ライブラリには以下のものがある。

  • JDBC DatabaseMetaData

    • DBのメタデータの読み取りにJDBCのAPIを使用している。

  • Thymeleaf

    • JPA Entityクラスのjavaソースコードを生成するのにテンプレートエンジンとしてThymeleafを使用している。

データアーキテクチャ

物理データモデル

jpa-entity-generatorプロジェクトはテスト用DBを内包している。 開発者は動作確認・テストの際にこのテスト用DBをリバースエンジニアリングしてJPA Entityを生成する。

テスト用DBのデータモデルを以下に示す。

テスト用DBにはJPA Entityの生成仕様を以下の観点でテストするためのテーブルが定義されている。

  • データ型

    • カラムのDBのデータ型がどのJavaのデータ型にマッピングされるか

  • 単独主キー

    • テーブルが単独主キーを持つ場合に、IDフィールドを正しく持つJPA Entityが生成されるか

  • 複合主キー

    • テーブルが複合主キーを持つ場合に、EmbeddedIdクラスとそのフィールドを正しく持つJPA Entityが生成されるか

  • リレーション

    • テーブル間のリレーションを正しく持つJPA Entityが生成されるか(OneToOne、OneToMany、ManyToOne、ManyToMany)

テスト用DBの詳細な構成は テスト用DBの構成 を参照。
