application architecture

Application behavior

jpa-entity-generator is a batch application. When JPA-entity-Generator is executed, metadata (table definitions, column definitions, key information, etc.) is read from the DB, and a JPA Entity Java source file for accessing them is generated. The execution forms provided by jpa-entity-generator are a Java command (Uber jar)

and a Maven Plugin.

The behavior of jpa-entity-generator is shown below.

Figure 1. Application behavior

  1. A user executes jpa-entity-generator.

  2. The

    jpa-entity-generator reads a configuration file. The user defines settings related to connection information to the DB and generation of JPA entities in a settings file in advance.

  3. The jpa-entity-generator reads metadata from DB.

  4. The jpa-entity-generator outputs a JPA entity Java source file based on metadata.

The main Java functions and third-party libraries used by jpa-entity-generator are as follows.

  • JDBC databaseMetaData

    • The JDBC API is used to read DB metadata.

  • Thymeleaf

    • Thymeleaf is used as a template engine to generate Java source code for JPA entity classes.

data architecture

physical data model

The

jpa-entity-generator project contains a test DB. Developers reverse engineer this test DB to generate JPA entities during operation confirmation/testing

.

The data model of the test DB is shown below.

A table for testing JPA Entity generation specifications from the following viewpoints is defined in the test DB.

  • data type

    • Which Java data type does the column DB data type map to

  • Single primary key

    • When a table has a single primary key, is a JPA Entity with the correct ID field generated?

  • composite primary key

    • If the table has a composite primary key, will the embeddedID class and a JPA Entity with the correct fields be generated

  • relations

    • Are JPA entities with correct relationships between tables generated (oneToOne, oneToMany, manyToOne, manyToMany)

For the detailed configuration of the test DB, see Test DB Configuration.
